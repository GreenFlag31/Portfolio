Code Maintenance

The first step while facing a bug is to identify the source of the problem. Identify a bug is a key step in problem resolution: we can spend a lot of time identifying a simple problem. In this article, I have selected the main tools available in Chrome (concerning medium and advanced categories) based on my experience. Similar tools can be found on other browsers, only the name might change.

Basic

Basic techniques are adapted while facing a simple problem. For exemple, the value of a variable is not what we expected.

Console.log( ) will simply print out a value in the console. This might be put everywhere in the code. Simply does not mean useless: sometimes, this is enough to identify the source of a problem.

Console.table( ) is adapted for multiple values, formatted in a table form, with rows and columns, like an Excel spreadsheet.

Console.time( ) is useful for performance computation, showing the time consumed by the code. If you code is already optimised, you will rarely need it.

Most used technique in this category will be Console.log( ). Careful though: this won't be adapted if you use it to print out the value of arrays or objects, since value in objects are passed by reference (they share the same address in memory). In other words, the value of an object might be updated after your Console.log( ), and the console will show you the most recent value. You might end up with an erroneous identification of the problem. In this case, more advanced techniques should be prefered.

Medium

Debugger is a breakpoint that will stop the execution of the code.
According to my experience, in a more advanced category, Debugger is the most useful tool. Having the execution of the code beeing stopped at a certain point allow you to watch all the expressions you want. This is an adapted technique to check the value of an object. This useful source presents you the different steps of code execution.

Advanced

Advanced are less used but useful techniques.
Watch is available in the sources tab of the Chrome console. According to me, this is mainly used in combination with a debugger breakpoint. It allows you to create a 'spy' that will print out the value of a selected variable.

Live Expression is an interesting technique that tracks in realtime the value of an expression and is available in the console tab. Just type the variable name you want to watch, and the value will be updated automatically, saving you many console.log( ). Scope of variables are respected though, so a block-scoped variable will be globally undefined.

Of course, all these techniques can be combined to produce the best result. 'Advanced category' does not mean that it is better than a simple Console.log( ): using the right tool in the right situation is what matters.
