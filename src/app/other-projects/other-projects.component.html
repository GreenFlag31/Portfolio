Calculator

Dec. 2022

Calculator is a one page project. This project presents a basic, non scientific calculator, to handle simple computations. Simple does not mean easy: a lot of user input validations had to be done.

Part of a Front-End training, the purpose of this project is to handle multiple exceptions that can occur. Focus is put mainly on Javascript and the structure of the code. This project also offers a correct user experience, supporting mouse and keyboard inputs.

Live Code

What have I learned ?

Deeper understanding of Javascript

Focus on user experience

Handle multiple exceptions, validation of user inputs

Code structure, files separated into modules to keep architecture optimal

Use of JSDoc to document code functions

What could be improved ?

For high structured projects, use of Object Oriented Programming might be preferable

Use of Reactive Programming might be preferable to maintain variables in a flow, saving the need of manually updating them

Game

Dec. 2022

Game is a one page project and is meant to be played and started in the console. This project is a classical paper-scissors-rock game with a touch of RPG fantasy.

Part of a Front-End training, the purpose of this project is to focus solely on Javascript. Enrichted with questions about Javascript, this game proposes a ranking system, storing the data in the sessionStorage.

Validation of user inputs

This portfolio

April 2023

This portfolio is a single page application (SPA) build with Angular, animated with route transitions.

The goal of this website is - obviously :) - to showcase different projects I made and practice further Angular. A relative complex point has been to handle the different route animations with the attached scroll behaviour. Attention has been put towards a smooth user experience. Recommended practices by Angular are followed. Edit June 2023: added features to the carousel on the landing page to improve user experience, supporting mouse and touch event.
